\doxysubsection{Detailed Description}
A data structure to maintain certain information related to a trajectory change point (TCP). Typically, this information is associated with a \mbox{\hyperlink{classlarcfm_1_1_nav_point}{Nav\+Point}} through a \mbox{\hyperlink{classlarcfm_1_1_plan}{Plan}}. The major purpose of this class is to store the relevant TCP information (e.\+g., status information, acceleration values, etc.).

Possible types of points. ~\newline
 
\begin{DoxyItemize}
\item \char`\"{}\+Orig\char`\"{} are from the original plan with no change 
\item \char`\"{}\+Virtual\char`\"{} are temporary points, generally interpolated points (long legs are broken into segments). Virtual points will be silently over-\/written, and never register as overlapping with other points. ~\newline
 Virtual points may be deleted unexpectedly. Not for general use. Virtual points are used for to conflict detection and not expected to be returned to the pilot or plane\textquotesingle{}s automation. 
\item \char`\"{}\+Alt\+Preserve\char`\"{} are marked points used in trajectory generation (\mbox{\hyperlink{classlarcfm_1_1_traj_gen_a76ac1994430b61a4c4ff0203e81048e6}{Traj\+Gen.\+make\+Kinematic\+Plan()}}) 
\end{DoxyItemize}

Each point can be a turn point (essentially a horizontal acceleration point), a ground speed acceleration point, or a vertical speed acceleration point. These points are indicated by either being the beginning of an acceleration zone or the end of an acceleration zone. Thus, there are methods to indicate the beginning of a turn (\mbox{\hyperlink{classlarcfm_1_1_tcp_data_a805732b3940471d62a27f4871217f066}{is\+BOT}}), or the end of the a turn (\mbox{\hyperlink{classlarcfm_1_1_tcp_data_afc4ba1bec6879cca07fd761e18f0fd32}{is\+EOT}}), similarly for ground speed of vertical speed.

Creates a new point that is the merger of the this point and the given point. Assumes that \mbox{\hyperlink{classlarcfm_1_1_tcp_data_a2df565a2453b2789ebdfa881beb729e2}{mergeable()}} on the two points would return true, if not, then approximately the original point is returned. 

Merging rules\+: 
\begin{DoxyItemize}
\item merge\+TCPInfo() is commutative 
\item If both points are linear points, then the points are merged. 
\item If one point is a linear point (aka, not a TCP), then the TCP info (velocity and acceleration) from the TCP point is used. 
\item If both points are TCP (aka acceleration points) of different types (Trk, Gs, Vs), then the points should merge without an issue 
\item If both points are TCP (aka acceleration points) of the same type (Trk, Gs, or Vs), but one is a beginning point and one is an ending point, then the resulting point should be a combined point (e.\+g., EOTBOT), and the TCP information should be the information from the beginning point. 
\item If both points are TCP \char`\"{}begin\char`\"{} points, then a message is provided if they have different source positions
