\doxysubsection{Detailed Description}
This class implements the algorithm for conflict detection between an ownship and a traffic aircraft, where the trajectories of both aircraft are modeled with intent information. 

This class can be used two ways\+: through an object or through a static method. The static method approach has the advantage that an object does not need to be created. The object approach has the advantage that more information is available. Specifically, the start and end times for every conflict are available in the object approach. 

When using the object method, all method \mbox{\hyperlink{classlarcfm_1_1_c_d_i_i_a4e4ed738c1bceca08aee28f07f6e619e}{get\+Time\+In()}}, etc, assume that the \mbox{\hyperlink{classlarcfm_1_1_c_d_i_i_a19c9a0cc561938673e1bab857185b8b8}{detection()}} method has been called first.

Usage\+: CDII.\+detection(ownship,traffic,\+B,\+T)

If there are long legs in the ownship \mbox{\hyperlink{classlarcfm_1_1_plan}{Plan}}, then it is prudent to call

\mbox{\hyperlink{classlarcfm_1_1_plan_util_ace8199b62c773b20aac179667b35a402}{Plan\+Util.\+interpolate\+Virtuals}}(ownship, \mbox{\hyperlink{classlarcfm_1_1_units_abbcc1b16b94f83ab9568d16a9d2dbdf7}{Units.\+from}}(\char`\"{}\+NM\char`\"{},0.\+1) ,0 , Tinfinity);

before the detection call.
