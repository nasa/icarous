\doxysubsection{Detailed Description}
This class defines the conversion factors for various units. Base units are defined for the seven base dimensions\+: length, mass, time, electric current, thermodynamic temperature, amount of substance, and luminous intensity. Together these seven units form a coherent set of units. If all quantities in a program are defined in terms of these seven coherent units, then no unit conversions are required within the program. Unit conversions are only needed when specifying and outputting quantities. This class is designed to facilitate that type of use. 

For example, note there are no unit conversions in the equations\+: 


\begin{DoxyPre}
double one\_mile = \mbox{\hyperlink{classlarcfm_1_1_units_abbcc1b16b94f83ab9568d16a9d2dbdf7}{Units.from}}("{}mi"{}, 1.0);
double one\_km = \mbox{\hyperlink{classlarcfm_1_1_units_abbcc1b16b94f83ab9568d16a9d2dbdf7}{Units.from}}("{}km"{}, 1.0);
double time = \mbox{\hyperlink{classlarcfm_1_1_units_abbcc1b16b94f83ab9568d16a9d2dbdf7}{Units.from}}("{}hour"{}, 1.0);

double distance = one\_mile + one\_km;
double speed = distance / time;

System.out.println("{}The distance should be 2.6 in kilometers and is "{} + \mbox{\hyperlink{classlarcfm_1_1_units_aac2f400e7a0ee3098f8500245efdd886}{Units.to}}("{}km"{}, distance)); 
System.out.println("{}The distance should be 1.6 in miles and is "{} + \mbox{\hyperlink{classlarcfm_1_1_units_aac2f400e7a0ee3098f8500245efdd886}{Units.to}}("{}mi"{}, distance)); 
System.out.println("{}The distance should be 8560 in feet and is "{} + \mbox{\hyperlink{classlarcfm_1_1_units_aac2f400e7a0ee3098f8500245efdd886}{Units.to}}("{}ft"{}, distance)); 
System.out.println("{}The speed should be 1.6 in miles/hour and is "{} + \mbox{\hyperlink{classlarcfm_1_1_units_aac2f400e7a0ee3098f8500245efdd886}{Units.to}}("{}mph"{}, speed)); 
System.out.println("{}The speed should be 0.72 in m/s and is "{} + \mbox{\hyperlink{classlarcfm_1_1_units_aac2f400e7a0ee3098f8500245efdd886}{Units.to}}("{}m/s"{}, speed)); 
\end{DoxyPre}



\begin{DoxyItemize}
\item One important consideration when using the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class is that the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class performs {\bfseries{NO}} consistency checks. The \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class trusts that the user will use units with a consistent dimension. This was a deliberate design decision because of the performance penalty associated with checking consistency. An example of this inconsistency is\+: 
\end{DoxyItemize}


\begin{DoxyPre}
double one\_mile = \mbox{\hyperlink{classlarcfm_1_1_units_abbcc1b16b94f83ab9568d16a9d2dbdf7}{Units.from}}("{}mile"{}, 1.0);
System.out.println("{}One mile in liters is "{} + \mbox{\hyperlink{classlarcfm_1_1_units_aac2f400e7a0ee3098f8500245efdd886}{Units.to}}("{}liter"{}, one\_mile));
\end{DoxyPre}


Obviously this makes no sense, but the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class will not flag this as an error; be careful. 

One can determine if two units are compatible by using the \mbox{\hyperlink{classlarcfm_1_1_units_a068af7834d86557e874cd4f7518e186c}{is\+Compatible}} method. \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} are considered compatible if they have the same dimensionality. For instance, the unit \char`\"{}meter\char`\"{} is compatible with the unit \char`\"{}foot\char`\"{} since they are both units of distance. 


\begin{DoxyPre}
if (!isCompatible("{}m"{}, "{}L"{})) \{
    System.out.println("{}'m' and 'L' are not compatible units!"{});
\}
if (isCompatible("{}ft"{}, "{}m"{})) \{
    System.out.println("{}'ft' and 'm' are compatible units!"{});
\}
\end{DoxyPre}



\begin{DoxyItemize}
\item One can always correctly convert between compatible units. To get the list of units compatible with a given unit use the \mbox{\hyperlink{}{get\+Compatible\+Units}} method. 
\end{DoxyItemize}

The user can also create their own composite units with the \mbox{\hyperlink{}{add\+Unit(\+String, double, String)}} method. For example a {\itshape fathom} unit could be defined as\+: 

{\ttfamily Units.\+add\+Unit(\char`\"{}fathom\char`\"{}, 6.\+0, \char`\"{}foot\char`\"{}); }

This method means, define the unit called \char`\"{}fathom\char`\"{}, where one fathom is equal to 6.\+0 feet. This method also states that the new \char`\"{}fathom\char`\"{} unit is compatible with the unit of \char`\"{}foot.\char`\"{} 

To create a new composite unit that has no compatible units, use the \mbox{\hyperlink{}{add\+Unit(\+String, double)}} method. For example a {\itshape linear density} unit could be defined as\+: 

{\ttfamily Units.\+add\+Unit(\char`\"{}kg/m\char`\"{}, \mbox{\hyperlink{classlarcfm_1_1_units_a0217219988003ff26f58d2dec08cbd58}{Units.\+kg}} / Units.\+meter);}

\doxysubsubsection*{Special \mbox{\hyperlink{classlarcfm_1_1_units}{Units}}\+:}


\begin{DoxyItemize}
\item unitless -\/ a name to represent that the quantity has no units. Ratios and percentages are typical examples of unitless quantities.


\item unspecified -\/ a name to represent that the units of the quantity that are unknown. The real distinction between a {\itshape unitless} quantity and an {\itshape unspecified} quantity is the notion of units compatibility. Unitless is not compatible with any other unit, and unspecified is is compatible with all units.  
\end{DoxyItemize}

\doxysubsubsection*{Usage Notes\+:}


\begin{DoxyItemize}
\item 

An alternate way to use the units class is to use constant factors instead of string representations. For example\+: 




\begin{DoxyPre}
double one\_mile = \mbox{\hyperlink{classlarcfm_1_1_units_abbcc1b16b94f83ab9568d16a9d2dbdf7}{Units.from}}(Units.mile, 1.0);
double one\_km = \mbox{\hyperlink{classlarcfm_1_1_units_abbcc1b16b94f83ab9568d16a9d2dbdf7}{Units.from}}(Units.kilometer, 1.0);
double time = \mbox{\hyperlink{classlarcfm_1_1_units_abbcc1b16b94f83ab9568d16a9d2dbdf7}{Units.from}}(\mbox{\hyperlink{classlarcfm_1_1_units_a94f47b552859c673fd991fb6dc8e3fc9}{Units.hour}}, 1.0);
\end{DoxyPre}




The advantage of this approach is that it is slightly more efficient since a lookup of the string value does not need to be performed. Since this operation only occurs when values are input, it is questionable how much time is really saved.




\item 

The above examples (using either strings or factors) are the preferred use of the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class; however, this notation can become cumbersome--especially when specifying values inside code. An alternate use of the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class is to multiply by the conversion factor when specifying constants. For example\+: 




\begin{DoxyPre}
double one\_mile = 1.0 * Units.mile;
System.out.println("{}One mile in kilometers is "{} + \mbox{\hyperlink{classlarcfm_1_1_units_aac2f400e7a0ee3098f8500245efdd886}{Units.to}}("{}km"{}, one\_mile));
\end{DoxyPre}




The "{}to"{} and "{}from"{} methods are preferred, because some conversions are not simply multiplying a factor, but involve an offset also. Those that involve an offset (currently only degreeC and degreeF), must use the \char`\"{}to\char`\"{} and \char`\"{}from\char`\"{} methods. Forms such as {\ttfamily double temp = 32.\+0 $\ast$ degreeF;} are {\itshape always} wrong.




\item The \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class does not define a \char`\"{}pound\char`\"{}. The problem is that in most official standards documents, a pound is a unit of mass; however, in most science and engineering texts, a pound is a unit of force. We avoid the problem by not defining a pound at all, and instead defining a pound\+\_\+mass (lbm) and a pound\+\_\+force (lbf). 


\item There are many more units defined as strings in the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class than the number defined as class fields in the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class. To get a complete list of all units defined as a strings call the \mbox{\hyperlink{}{get\+All\+Units()}} method. 


\item 

The internal unit for angles is always radians. The SI convention is followed by calling this a dimensionless derived unit, defined as m / m. 





Since angles are always represented in radians, a user of the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class does not need to perform unit conversions to get radians for trigonometric functions. Expressions like 

{\ttfamily a = Math.\+cos(\mbox{\hyperlink{classlarcfm_1_1_units_abbcc1b16b94f83ab9568d16a9d2dbdf7}{Units.\+from}}(\char`\"{}radian\char`\"{}, pi\+\_\+over\+\_\+2));} 

are not necessary, but are still valid. Expressions that use degrees still need units specified, for example\+: 

{\ttfamily double right\+\_\+angle = \mbox{\hyperlink{classlarcfm_1_1_units_abbcc1b16b94f83ab9568d16a9d2dbdf7}{Units.\+from}}(\char`\"{}degree\char`\"{}, 90.\+0);}~\newline
 {\ttfamily a = Math.\+cos(right\+\_\+angle);} 


\item The units for thermodynamic temperature are degreeC, degreeF, K, and degreeR, representing Celsius, Farenheit, Kelvin, and Rankin. 


\item SI defines the unit of frequency as the reciprocal second and gives this quantity the name {\itshape hertz} and the symbol {\itshape Hz}. The \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class defines these quantities as \char`\"{}hertz\char`\"{} and \char`\"{}\+Hz\char`\"{}. Some disciplines use the term {\itshape frequency} to indicate an angular velocity. Specifically, the notion in dynamic systems of {\itshape natural frequency}, often abbreviated with the Greek symbol lower-\/case omega (\&\#969;), is actually an angular velocity. In the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class, the appropriate symbol for angular velocity is \char`\"{}radian\+\_\+per\+\_\+second\char`\"{} or \char`\"{}degree\+\_\+per\+\_\+second\char`\"{}. However, since quantities like {\itshape natural frequency} have frequency in their name, they are often, incorrectly, expressed in \char`\"{}hertz.\char`\"{} To accomodate this type of usage, the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class defines a unit \char`\"{}hertz\+Angular\char`\"{}. \char`\"{}hertz\char`\"{} and \char`\"{}hertz\+Angular\char`\"{} are {\bfseries{NOT}} compatible with each other, the first is a frequency and the second is an angular velocity. 


\item Both joules and newton-\/meters are defined. These units are so similar that they could be considered the same. They are kept separate because joules are used to represent energy and newton-\/meters are used to represent torque. Energy is a scalar quantity and torque is a vector. It is rare that they are interchanged. The US Customary unit for energy is the foot-\/pound (ft-\/lbf), and the unit for torque is the pound-\/foot (lbf-\/ft). This last name is our invention. 


\item Both names {\itshape metre} and {\itshape meter} and {\itshape litre} and {\itshape liter} are defined. Each pair is identical. In the approved English language version of the SI standard, only the terms \char`\"{}metre\char`\"{} and \char`\"{}litre\char`\"{} are defined. However, in the US version of this document, prepared by the National Institute of Standards and Technology (Special Pub. 330) using current US government policy, the only defined terms are \char`\"{}meter\char`\"{} and \char`\"{}liter.\char`\"{} The \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class defines both; however, the reader will observe a bias in the documentation in favor of the US terms. 
\end{DoxyItemize}

\doxysubsubsection*{Special \mbox{\hyperlink{classlarcfm_1_1_constants}{Constants}}\+:}


\begin{DoxyItemize}
\item 

The \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class defines {\itshape gn} as the adopted physical constant of gravity. It is given (out of place) in the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class, due to the fact that the definition of the fundamental US customary unit of mass (slug) depends on this quantity. 





This quantity, \mbox{\hyperlink{classlarcfm_1_1_units_aa2bb8b0caabbeeddf2ac551368e33d01}{Units.\+gn}}, is defined in IEEE/\+ASTM SI 10-\/1997 as (exactly) 9.\+80665 m/s$^\wedge$2 \mbox{[}page 25\mbox{]}. The 1962 U.\+S. Standard Atmosphere \mbox{[}page 4\mbox{]} calls this quantity \char`\"{}standard sea-\/level gravity\char`\"{} and describes it as the acceleration due to the combined effects of gravity and the Earth\textquotesingle{}s rotation (i.\+e. the centrifugal relief) at geodetic latitude 45 degrees. This constant was adopted in the US in 1935. 





The latitude of 45 degrees was chosen because this is the standard latitude chosen by the World Meteorological Organization to calibrate barometers. As a note, this value is the solution to Lambert\textquotesingle{}s equation (sea-\/level gravity as a function of latitude), when the latitude is 45 degrees 32 minutes and 33 seconds.




\item The \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class defines \mbox{\hyperlink{classlarcfm_1_1_units_a7e9fe90b66bfbe5652f316a89c4d22a8}{Units.\+P0}} as the adopted physical constant of standard atmosphere. This quantity equals 101325 Pa. This definition is added to the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class because the definition of an atmosphere unit (\char`\"{}atm\char`\"{}) depends on it. This definition comes from NIST Special Publication 330, the International System of \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} (SI), 1991 edition, p15. 
\end{DoxyItemize}

\doxysubsubsection*{Implementation Notes\+:}


\begin{DoxyItemize}
\item The \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class uses the SI system (with base units\+: meter \mbox{[}distance\mbox{]}, kilogram \mbox{[}mass\mbox{]}, second \mbox{[}time interval\mbox{]}, ampere \mbox{[}electric current\mbox{]}, kelvin \mbox{[}thermodynamic temperature\mbox{]}, mole \mbox{[}amount of substance\mbox{]}, and candela \mbox{[}luminous intensity\mbox{]}) as a coherent set of base units (internal units). 



Please remember the user of the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class should never {\itshape need} to know how the variables are stored internally. The user need only remember to specifically state the units on input and output and any conversions will occur automatically. 


\item The purpose of this class is geared to modeling large engineered systems such as airplanes. As such, many units useful in other disciplines are not defined (such as the \char`\"{}carat\char`\"{} or a \char`\"{}bushel\char`\"{}). However, they may be defined in future versions of this class. 



Please note that since the complete set of seven base units are defined, any other units needed may be defined by the user with the \mbox{\hyperlink{}{add\+Unit()}} method.


\item Much care was used in defining each of these conversion factors. The full precision from appropriate standards documents was used. However, some units are defined as the arithmetic relationship between \textquotesingle{}more fundamental\textquotesingle{} quantities. In these cases, the conversion factors are limited to the resolution of double precision operations. For most engineering purposes (see the point above) this is more than good enough; however, someone working with very precise quantities (\texorpdfstring{$>$}{>} 12 significant digits), should be aware of the possibility of differences.


\item The names of many of the conversion factors violate standard naming conventions for class variables. This was deemed acceptable since these names follow the standard capitalization conventions from SI or other standards bodies. 


\item The \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class is not {\itshape immutable}. However modification of this class is limited. Specifically, there is no mechanism to {\itshape delete} a unit from the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class, nor is there any ability to {\itshape change} a conversion factor of a unit. These restrictions allow the user to be confident that once a unit is in the \mbox{\hyperlink{classlarcfm_1_1_units}{Units}} class, its definition will never change. The unit {\bfseries{A}} will always be an ampere, it will never become an angstrom.


\end{DoxyItemize}
