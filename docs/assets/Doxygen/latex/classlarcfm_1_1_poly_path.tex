\doxysubsection{Detailed Description}
Polygons are arranged in \char`\"{}paths\char`\"{} similar to aircraft plans (and, in fact, this object can produce one -- or more -- faux-\/aircraft plans that approximate the area in question). The polygon is allowed to change shape as it travels along the path, but the number of vertices must remain constant. The shape at non-\/specified time points is linearly interpolated from the neighboring points. ~\newline


(Note that the default mode has changed from MORPHING to AVG\+\_\+\+VEL.)

Paths can have one of several modes\+: AVG\+\_\+\+VEL (the default)\+: Velocities are calculated between the average points of consecutive steps. Polygons do not need to have the same number of vertices at each step. The path starts to exist at the first time and stops existing at the last time. \mbox{\hyperlink{classlarcfm_1_1_vertical}{Vertical}} speed is based on the bottom of the steps. In the case of a single-\/step path, velocity is zero and time is ignored\+: the polygon always exists statically at that location. The time interval the path exists is either \mbox{[}t1,t2\mbox{]} or \mbox{[}0,+inf\mbox{]} (for 1-\/step paths) MORPHING\+: all polygons must have the same number of vertices. The path starts to exist at the first time and stops existing at the last time. \mbox{\hyperlink{classlarcfm_1_1_vertical}{Vertical}} speed is based on the bottom of the steps. In the case of a single-\/step path, velocity is zero and time is ignored\+: the polygon always exists statically at that location. While different modes can mimic many of the same behaviors, there are some internal optimizations when processing certain polygon modes.
