\doxysubsection{Detailed Description}
A class to parse files with input that are in columns separated by commas, spaces, or tabs. 

General notes on the file\+: 
\begin{DoxyItemize}
\item All blank lines are skipped 
\item All lines that begin with a \textquotesingle{}\#\textquotesingle{} are skipped 
\item The first section of the file contains parameters. This parameter section must be before the data 
\item The next section is the main data of the file. It contains columns of data. 
\item The file can be read with case sensitive or insensitive strings. Case sensitivity applies to header names and parameter keys (if case insensitive, these will be converted to lower case internally) 
\item The first line after the column name heading may be an optional line specifying nonstandard units for each column. These are represented as \mbox{[}nmi\mbox{]}, \mbox{[}fpm\mbox{]}, \mbox{[}deg\mbox{]}, etc. If so defined, then values stored will be converted from these units into internal units. ~\newline
 Otherwise it is assumed that values are already in internal units. ~\newline
 If the default units as understood by the end user are not internal units (for example using degrees for latitude instead of radians), then further processing of the stored data may be necessary upon retrieval. Use \mbox{[}unspecified\mbox{]} for any column that has no applicable units (e.\+g. strings or booleans). A dash (-\/) in the units field will be interpreted as \char`\"{}unitless\char`\"{}. 
\end{DoxyItemize}

Notes on the columns of data 
\begin{DoxyItemize}
\item The columns of data may be separated by any number of commas, spaces, or tabs (but see \mbox{\hyperlink{classlarcfm_1_1_separated_input_a0b44757bf64faa3a507039757e246b14}{set\+Column\+Delimiters}}) 
\item The first line of the columns of data is considered the \char`\"{}headings.\char`\"{} These are strings which can be accessed with the \mbox{\hyperlink{classlarcfm_1_1_separated_input_aaab6d1849541c6cd41e93504ddd88320}{get\+Heading()}} method. 
\item The next line is optional. It contains the a string representing the units for a column. A line is considered a line of units if at least half of the fields read in are interpreted as valid units. 
\item The columns of data can be strings or double-\/precision floating point values 
\item If the values are doubles, then they are assumed to be in internal units. If a unit type is specified, their values will be converted into internal units and stored. The end user is responsible for converting values if the unspecified default units are something other than the internal ones. 
\item This format essentially follows RFC 4180 (when using commas) ~\newline
 
\end{DoxyItemize}

Parameters are essentially a \char`\"{}key/value\char`\"{} pair. Once an parameter is entered (either from the set\+Parameter method or from the file), then that parameter name is reserved. If any future updates to the parameter (either from the set\+Parameter method or the file), then the previous value will be overwritten. Parameters have the following rules\+: 
\begin{DoxyItemize}
\item Parameters have the basic form \char`\"{}key = value\char`\"{} 
\item Parameter keys can be any combination of letters and numbers. 
\item There must be spaces around the equals sign 
\item The value may be a string, a double-\/precision value, or a boolean value. 
\item If the value is a double precision value, then units may be added. ~\newline
 For example, \char`\"{}param = 10 \mbox{[}nmi\mbox{]}\char`\"{} 
\item The units are optional. 
\item Boolean values are represented by T, F, true, false, TRUE, or FALSE. 
\item If a parameter has not been set and it is read, something will be returned. It is not defined what that something is. To avoid this, check is a parameter exists with the contains\+Parameter() method. 
\end{DoxyItemize}
